EM&M Git2 Repo Server fail-over
powered by Git, Apache, and CentOS

Welcome to the Event Management & Mediation (EM&M) Repository System. This system is based on the Git distributed version control system. It is managed with Gitolite.

This repository system encompasses two servers, this fail-over server and an primary server.

NOTE: This server is read-only, updated daily (M-F), with a backup/restore from the EMM Git1 server.

If you are new to Git:

The official Git site is git-scm.com, cover chapters 2 and 3 at minimum under the Documentation link. Another good source for Git usage and introductions is Notes on Git. Your EM&M Configuration Management team also maintains the EM&M Git Users Guide. In order to deliver EM&M deployments, you'll need to follow this user guide.

Deployment WorkFlow and Documentation

The development work you do for EM&M is managed with Change Records (CR)s. These CRs come in the form of Service Requests (SR00NNNNNN) which are opened for production problems in a Remedy system, User Stories (USNNNNNN) which are managed in the Rally system, and legacy Change Records (CR-NNNN) which are managed with an EM&M Sharepoint site.

EM&M Git repos are application modules that are maintained by developers in development environments for promotion to all other EM&M environments (QA, Test labs, Production). These promotions are done through this repository system, managed by the EM&M CM and EPA teams per instructions provided by development.

EM&M Deployments are managed with the JIRA CEMPCM project where CRs and Git repos are associated in the titles (summaries) of each JIRA CEMPCM project ticket. A Dev Queue dashboard is available for EM&M developers to manage their work.

A Comcast wiki maintained with Confluence is utilized for EM&M documentation. A EM&M CM Portal is maintained to provide a view to all things Comcast EM&M including legacy cemp, Comcast services, and other documentation locations.

About Git:

Git is a distributed revision control and source code management (SCM) system with an emphasis on speed, data integrity,and support for distributed, non-linear workflows. Git was initially designed and developed by Linus Torvalds for Linux kernel development in 2005, and has since become the most widely adopted version control system for software development.

As with most other distributed revision control systems, and unlike most client-server systems, every Git working directory is a full-fledged repository with complete history and full version tracking capabilities, independent of network access or a central server. Like the Linux kernel, Git is free software distributed under the terms of the GNU General Public License version 2.

About Apache:

The Apache HTTP Server Project is an effort to develop and maintain an open-source HTTP server for modern operating systems including UNIX and Windows NT. The goal of this project is to provide a secure, efficient and extensible server that provides HTTP services in sync with the current HTTP standards.

About CentOS:

The Community ENTerprise Operating System (CentOS) is an Enterprise-class Linux Distribution derived from sources freely provided to the public by a prominent North American Enterprise Linux vendor. CentOS conforms fully with the upstream vendors redistribution policy and aims to be 100% binary compatible. (CentOS mainly changes packages to remove upstream vendor branding and artwork.) The CentOS Project is the organization that builds CentOS.
