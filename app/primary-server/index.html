Code and Deployment Workflow

Development work is initiated with Change Records (CR)s. These CRs come in the form of Operations support tickets, production problems tracked in JIRA project Operations, User Stories (USNNNNNN) which are managed in the Rally system, and legacy Change Records (CR-NNNN) which are managed with an Sharepoint site. Coordination of module development and deployment is managed with JIRA CEMPCM Project tickets.

Git repos are modules modified by developers in development environments for promotion to all other environments (QA, Integration, Stage, Production). This is the JIRA Workflow.

Deployments are managed using the JIRA CEMPCM project where CRs and Git repos are associated in the titles (summaries) of each JIRA CEMPCM project ticket. A Dev Queue dashboard is available for developers to manage their work.

A wiki maintained with Confluence is utilized documentation. A CM Portal is maintained to provide a view to all things supported org including legacy cemp, Co services, and other documentation locations.
