Welcome to the Event Management & Mediation (EM&M) Repository System. This system is based on the Git distributed version control system. It is managed with Gitolite.

This repository system encompasses two servers, this Primary server and the fail-over server.

Code and Deployment Workflow

Development work for EM&M is initiated with Change Records (CR)s. These CRs come in the form of EMM Operations support tickets (EMMLSUP-NNNN), production problems tracked in JIRA project EMM Operations, User Stories (USNNNNNN) which are managed in the Rally system, and legacy Change Records (CR-NNNN) which are managed with an EM&M Sharepoint site. Coordination of module development and deployment is managed with JIRA CEMPCM Project tickets.

EM&M Git repos are EM&M modules modified by developers in development environments for promotion to all other EM&M environments (QA, Integration, Stage, Production). This is the JIRA EM&M Workflow.

EM&M Deployments are managed using the JIRA CEMPCM project where CRs and Git repos are associated in the titles (summaries) of each JIRA CEMPCM project ticket. A Dev Queue dashboard is available for EM&M developers to manage their work.

A Comcast wiki maintained with Confluence is utilized for EM&M documentation. A EM&M CM Portal is maintained to provide a view to all things Comcast EM&M including legacy cemp, Comcast services, and other documentation locations.
